# Lab 01: Creating and Deploying a MongoDB Atlas Cluster

## 🎯 Objective
The goal of this lab was to set up a fully functional, cloud-hosted NoSQL database cluster using MongoDB Atlas. This forms the foundation for all future database administration tasks.

## 🛠️ Infrastructure Details
- **Cloud Provider:** AWS / Google Cloud (As per lab selection)
- **Region:** Mumbai (ap-south-1) - Low Latency 🇮🇳
- **Cluster Tier:** M0 (Free Tier - ideal for sandbox testing)
- **Replication:** 3-node replica set (High Availability)

## 🚀 Steps Performed
1. **Organization & Project Setup:** Created a dedicated project environment in Atlas.
2. **Cluster Configuration:** Selected the cloud provider and region with low-carbon emissions.
3. **Security Setup:** Automated the initial security configuration (IP Whitelisting & User creation).
4. **Sample Dataset:** Pre-loaded sample data for testing aggregation and queries.

## 💡 Key Learnings (Admin Perspective)
- Understanding the difference between Shared and Dedicated tiers.
- How Cloud Providers (AWS/GCP/Azure) manage the physical storage of our NoSQL data.
- Importance of selecting a region close to the users to reduce latency.
